== Objective

The goal of this analysis is to verify the alignment between identified domain problems, such as pain points and needs, and system features defined in the project system.

NOTE: All domain needs, pain points and system features have been extracted and derived from the Milestone 2 documentation.

== Domain Needs and Pain Points

=== Needs

Key needs identified from Section 1.2.2:

* N1 — Minimize effort/time required to track household items
* N2 — Maintain accurate understanding of available items
* N3 — Verify item availability before purchasing
* N4 — Shared household visibility of inventory
* N5 — Coordinate replenishment of shared items

Quality needs identified from Section 1.2.2:

* Usability: low-effort updates
* Reliability: accurate inventory state
* Performance: fast access to information
* Availability: usable under intermittent connectivity
* Privacy: controlled access per household
* Scalability: supports multiple users and items

=== Pain Points

Pain points identified from Section 1.2.1.4:

* P1 — Item entry is too time-consuming
* P2 — Inventory becomes outdated quickly
* P3 — Excessive or irrelevant notifications
* P4 — High pricing / unsuitable product-market fit for students
* P5 — Lack of effective shared household inventory support

== System Features

Key system capabilities identified from Sections 1.2.3.1 and 1.3.1:

* F1 — Fast item capture with minimal input effort
* F2 — Centralized household inventory view
* F3 — Shared inventory visibility and collaboration
* F4 — Item search and filtering (availability check)
* F5 — Low-noise notifications for restocking/expiration
* F6 — Offline support with later synchronization
* F7 — Budget tracking and spending summaries
* F8 — Categorization by room, type, and attributes
* F9 — Shared responsibility tracking (who updated what)

== Gaps and Mismatches

1. Lack of explicit task assignment for replenishment (N5):
   - Current system tracks responsibility but does not enforce or guide action completion.

2. No mechanism ensuring long-term data freshness (P2):
   - Users can still forget to update inventory over time.

3. Pricing constraint (P4) is not addressed:
   - The system assumes adoption without considering affordability or feature gating.

== Recommendations for Improvement

=== Improve Replenishment Coordination (N5 gap)

Add feature “Restock Task System”: to allow users to assign replenishment tasks, include due dates and reminders, and implement track completion status

Benefit: Converts passive responsibility tracking into actionable workflow

=== Improve Inventory Freshness (P2 gap)

Add or enhance “Staleness Detection Mechanism”: flag items not updated in a long time, prompt lightweight confirmation (e.g., "still have this?"), and add optional auto-expiry rules for consumables

Benefit: Improves reliability of inventory data over time

=== Address Pricing Constraint (P4 gap)

Possible approaches:

* Ensure core functionality is fully free for students
* Avoid gating essential features behind premium tiers
* Consider institutional deployment model (e.g., university-supported tool)

Benefit: Improves accessibility and adoption in target demographic
