= Static vs Dynamic Classification of Home Inventory Phenomena

*AI Disclosure*: This document was developed with the assistance of an AI tool for structural guidance and a better understanding of lecture concepts. All classifications, interpretations, and final decisions were reviewed, validated, and finalized by the author.

This document applies the lecture concept of *static* and *dynamic* domain attributes to the Home Inventory system.  

Each identified domain phenomenon is classified as:

* *Static* - Does not change once defined.
* *Dynamic* - Can change over time.

Dynamic phenomena are further classified as:

* *Inert* - Changes naturally without system intervention.
* *Reactive* - Changes in response to external events.
* *Programmable* - Changes due to explicit system logic or user commands.

== Explanation of Key Distinctions

=== Static Attributes
Static attributes represent properties that remain constant once defined. +
Example: *Item ID* remains fixed for system integrity and traceability.

=== Dynamic - Inert
Inert attributes change due to the passage of time or physical processes. +
Example: *Expiration Status* changes as time advances, even without user interaction.

=== Dynamic - Reactive
Reactive attributes change in response to external actions or system events. +
Example: *Quantity* changes when a user consumes or adds inventory.

=== Dynamic - Programmable
Programmable attributes change according to system logic or user-controlled configuration. +
Example: *Low-Stock Notification* is triggered based on programmed threshold rules.
