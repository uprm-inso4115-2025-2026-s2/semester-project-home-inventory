== Introduction

This task focuses on identifying and defining the main reporting concepts in the Home Inventory system. The goal is to understand how reporting works in the domain, independent of specific implementation decisions.

Having clear reporting concepts helps ensure that any reports generated by the system are consistent, meaningful, and aligned with how household data is actually used and understood.

== Main Reporting Concepts

=== Household

A household is the main context where inventory and spending are managed. In this project, spending and reporting are tied to a household instead of to an individual only.

Attributes:

* household_id
* name

Importance:

* Reporting is usually done at the household level
* Spending and inventory data are shared within a household

=== Spending entry

A spending entry is a record of a financial transaction in the household, such as a purchase, refund, or adjustment.

Attributes:

* id
* household_id
* user_id
* amount
* category
* description
* incurred_on
* created_at

Importance:

* Represents the main source of financial data
* Used to understand how money is being spent over time

=== Spending category

A spending category is a grouping used to organize spending, such as groceries, cleaning, or personal care.

Attributes:

* category name or label

Importance:

* Helps organize spending into meaningful groups
* Makes it easier to analyze patterns in household expenses

Note:

* In the current schema, the `expenses` table stores category as text, even though there is also a `categories` table in the database

=== Budget cycle (period)

A budget cycle is the time period used to evaluate spending, such as weekly or monthly.

Examples:

* Month
* Week
* Custom period

Importance:

* Provides a time frame for understanding spending behavior
* Allows comparison of spending across different periods

=== Item type / inventory-related concepts

The domain also includes inventory concepts such as item type, item instances, and tracked inventory. These are relevant because spending is often related to acquiring and maintaining household items.

Importance:

* Connects financial data to physical items in the household
* Helps provide context for why spending occurs

== Relationships Between Concepts

The main relationships are:

* A household can have many spending entries
* A spending entry belongs to one household
* A spending entry may be associated with a spending category
* Spending entries are observed over a budget cycle
* Inventory-related concepts provide context for understanding spending

These relationships allow the system to represent how household spending is structured and interpreted.

== Aggregation Rules

Monthly spending:

* Group spending entries by household
* Group them again by month and year using `incurred_on`
* Sum the `amount` values

Category-based totals:

* Group spending entries by household and spending category
* Sum the `amount` values for each category

Household-level summaries:

* Combine spending within a given period
* Compare or interpret totals across time or categories

These aggregation rules describe how raw spending data can be interpreted into meaningful information.

== Conclusion

From this analysis, the main reporting concepts in the Home Inventory system are household, spending entry, spending category, and budget cycle. These concepts define how financial data is structured and understood in the domain.

By clearly defining these concepts and their relationships, it becomes easier to interpret household spending and generate consistent reports. This ensures that any reporting features built on top of this model reflect how the domain actually works, rather than being based only on technical design choices.
