== 1. Introduction

This questionnaire is designed to gather structured information from potential end-users of the {project}. Its goal is to capture real household management behaviors, identify friction points, and inform system requirements.

The questionnaire supports requirements acquisition by uncovering domain concepts, user needs, and potential functional and non-functional requirements.

Questions are organized into thematic sections to guide a focused and efficient data collection process.

*Estimated completion time:*
- 10–15 minutes (written)
- 15–25 minutes (facilitated interview)

[NOTE]
====
Participants may skip any question they are not comfortable answering.

Facilitators should encourage concrete, recent examples rather than general or idealized responses  
(e.g., “Tell me about the last time…” instead of “Do you usually…”).
====

---

== 2. Questionnaire Structure

The questionnaire is divided into the following sections:

- Participant Background
- Item Tracking and Categorization
- Expiration and Waste
- Low-Stock Detection and Restocking
- Shared Household Coordination
- Data Entry and Usability
- Priorities and Open Feedback

== 5. Usage of Collected Data

The collected data will be used to:

- Identify behavioral patterns
- Detect common pain points
- Validate domain assumptions

The insights will be:

- Translated into user stories
- Mapped to system features
- Used to refine requirements

This ensures traceability between user needs and implementation.
