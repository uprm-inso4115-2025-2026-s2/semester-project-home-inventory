== Testing

Testing is the main verification mechanism for this project. For Milestone 1, this section defines the verification approach, the main test types, and an initial set of representative test cases.

=== Verification Strategy

* *Code review:* all changes to `main` must go through a pull request reviewed by at least one teammate who did not author the change. Reviews check correctness against requirements, edge cases, terminology consistency, and whether tests were added or updated.
* *Automated tests:* unit, integration, and verification tests run on pull requests and before merge. At minimum, affected tests must pass before changes are merged.
* *Manual checks:* short repeatable smoke checks are performed after merges for stakeholder-visible behaviors such as item updates, alerts, and shared inventory consistency.

=== Test Types

* *Unit tests:* verify isolated rules such as threshold crossing, expiration handling, ownership changes, state transitions, and transaction creation.
* *Integration tests:* verify persistence, retrieval, synchronization, and automation behavior across connected components.
* *End-to-end tests:* verify complete workflows such as adding/editing items, triggering low-stock conditions, and generating alerts without duplication.
* *UI tests:* verify that key screens present the correct state and controls for milestone features.

=== State Model Coverage

Verification uses the same alert and state vocabulary defined elsewhere in the project:

* Stock-related states: `NORMAL` / `IN_STOCK`, `LOW_STOCK`, `OUT_OF_STOCK`
* Expiration-related states: `Active`, `Expiring`, `Expired`, `Renewed`, `Deleted`

Verification focuses on state transitions and deduplication behavior regardless of delivery mechanism.

=== Definition of Done

* The feature meets the mapped acceptance criteria for the target user story.
* Relevant automated tests are added or updated and pass before merge.
* Documentation is updated when terminology, rules, or domain behavior changes.
* No unresolved review issues remain for correctness, edge cases, or traceability.
