:toc-title: Table of Contents

== Overview

This document describes the structured pipeline used to process raw domain description units
into higher-level abstractions for the Home Inventory Management System. The pipeline moves
from unstructured observations and low-level entity lists toward a coherent set of named
domain concepts and a unified narrative, while surfacing inconsistencies and gaps for
analyst review.

The pipeline consists of four sequential stages:

[source]
----
[Stage 1: Ingest]
   Raw Domain Description Units
         ↓
[Stage 2: Concept Formation]
   Group low-level entities → named domain concepts
         ↓
[Stage 3: Narrative Rewrite]
   Replace entity lists with concept terminology
         ↓
[Stage 4: Inconsistency Detection]
   Flag P / not-P conflicts, gaps, and incompletenesses
         ↓
[Output: Analyst Review Package]
   Unified narrative + flagged issues
----

---

artifacts collected during domain acquisition. For this project the raw DDUs were gathered
from student surveys, informal household practices, and course domain analysis exercises.

== Stage 2 — Concept Formation: Abstraction Mapping

In this stage, low-level entity lists from the raw DDUs are grouped into named domain
concepts at a higher abstraction level. The goal is to identify the *kinds of things*
the domain talks about, independent of any specific implementation.

=== Concept Hierarchy

The following hierarchy shows how concepts relate to one another:

[source]
----
Household
├── Household Member (1 or more)
└── Inventory (reality)
    └── Household Item (many)
        ├── Item Attributes (expiration, quantity, price...)
        ├── Location (where stored)
        └── Availability State (OK / Low / Out / Expired)

Tracked Inventory Record  ← approximates → Inventory (reality)

Replenishment Action  ← triggered by → Availability State (Low / Out)
Expenditure          ← recorded after → Replenishment Action
Budget               ← governs → Expenditure

Alert / Notification ← triggered by → Availability State or Budget threshold
Coordination Failure ← arises from → Tracked Inventory Record diverging from reality
----

== Stage 3 — Narrative Rewrite

See: link:unified-domain-narrative.adoc[Unified Domain Narrative]

The unified narrative replaces all raw entity lists (DDU-10 through DDU-22) with their
corresponding concept names from Stage 2. All terminology is drawn exclusively from the
project's Ubiquitous Language dictionary
(`docs/2-descriptive/domain-description/terminology/terminology.adoc`).

---

== Stage 4 — Inconsistency Detection

See: link:inconsistency-detection.adoc[Inconsistency Detection Report]

The inconsistency detector scans all DDUs and existing domain documents for:

* *Direct P / not-P conflicts* — one unit asserts a property that another directly negates.
* *Scope conflicts* — a rule applies a constraint that another rule exempts.
* *Completeness gaps* — a concept is referenced but never fully defined.
* *Terminology drift* — the same real-world phenomenon described under two different names.
