== Core Inventory Database Entities and Attributes

This document defines the Core Inventory database entities and their attributes, including data types, nullability, and default values. The purpose of this document is to ensure consistency between the database schema, Supabase implementation, and application models across the team.

The entities defined here are based on the system schema discussed in task #15 and existing Core Inventory documentation.

=== User

Represents a system user who can manage one or more homes and track inventory items.

Attribute: id  
Data Type: Integer  
Nullable: NOT NULL  
Default: auto-generated  

Attribute: email  
Data Type: String  
Nullable: NOT NULL  
Default: —  

Attribute: name  
Data Type: String  
Nullable: NOT NULL  
Default: —  

Attribute: created_at  
Data Type: Timestamp (date+time)
Nullable: NOT NULL  
Default: current timestamp  


=== Owner

Represents the person associated with an item. Owners may or may not be system users (for example, roommates).

Attribute: id  
Data Type: Integer  
Nullable: NOT NULL  
Default: auto-generated  

Attribute: name  
Data Type: String  
Nullable: NOT NULL  
Default: —  

Attribute: created_at  
Data Type: Timestamp (date+time)
Nullable: NOT NULL  
Default: current timestamp  


=== Home

Represents a household environment that contains rooms and inventory items.

Attribute: id  
Data Type: Integer  
Nullable: NOT NULL  
Default: auto-generated  

Attribute: name  
Data Type: String  
Nullable: NOT NULL  
Default: —  

Attribute: created_at  
Data Type: Timestamp (date+time)
Nullable: NOT NULL  
Default: current timestamp  


=== Room

Represents a location within a home where items are stored (for example, kitchen, pantry, bathroom).

Attribute: id  
Data Type: Integer  
Nullable: NOT NULL  
Default: auto-generated  

Attribute: name  
Data Type: String  
Nullable: NOT NULL  
Default: —  

Attribute: home_id  
Data Type: Integer  
Nullable: NOT NULL  
Default: —  

Attribute: created_at  
Data Type: Timestamp (date+time)
Nullable: NOT NULL  
Default: current timestamp

Data Type: Integer  
Nullable: NOT NULL  
Default: auto-generated  

Attribute: name  
Data Type: String  
Nullable: NOT NULL  
Default: —  

Attribute: brand  
Data Type: String  
Nullable: Nullable  
Default: —  

Attribute: size  
Data Type: String  
Nullable: Nullable  
Default: —  

Attribute: variant  
Data Type: String  
Nullable: Nullable  
Default: —  

Attribute: barcode  
Data Type: String  
Nullable: Nullable  
Default: —  

Attribute: created_at  
Data Type: Timestamp (date+time)
Nullable: NOT NULL  
Default: current timestamp  


=== Item (Stock Entry)

Represents a specific instance of a product owned by a user or household, including quantity and expiration details.

Attribute: id  
Data Type: Integer  
Nullable: NOT NULL  
Default: auto-generated  

Attribute: product_id  
Data Type: Integer  
Nullable: NOT NULL  
Default: —  

Attribute: owner_id  
Data Type: Integer  
Nullable: NOT NULL  
Default: —  

Attribute: room_id  
Data Type: Integer  
Nullable: NOT NULL  
Default: —  

Attribute: quantity  
Data Type: Integer  
Nullable: NOT NULL  
Default: 1  

Attribute: purchase_date  
Data Type: Date  
Nullable: Nullable  
Default: —  

Attribute: expiration_date  
Data Type: Date  
Nullable: Nullable  
Default: —  

Attribute: notes  
Data Type: String  
Nullable: Nullable  
Default: —  

Attribute: created_at  
Data Type: Timestamp (date+time)
Nullable: NOT NULL  
Default: current timestamp  


=== User–Home Relationship

Represents the association between users and the homes they manage.

Attribute: user_id  
Data Type: Integer  
Nullable: NOT NULL  
Default: —  

Attribute: home_id  
Data Type: Integer  
Nullable: NOT NULL  
Default: —  

Attribute: role  
Data Type: String  
Nullable: NOT NULL  
Default: "member"  


=== User–Owner Relationship

Represents the association between users and owners they track inventory for.

Attribute: user_id  
Data Type: Integer  
Nullable: NOT NULL  
Default: —  

Attribute: owner_id  
Data Type: Integer  
Nullable: NOT NULL  
Default: —  


== Notes

* Data types are aligned with Supabase/PostgreSQL compatible types.
* Timestamp fields default to the current system time when records are created.
* Nullable fields represent optional information that may not always be available during item entry.
* This document serves as the reference for Supabase table implementation and application model alignment.
