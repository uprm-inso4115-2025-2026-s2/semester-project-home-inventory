== Main Contributions

=== Domain Description and Tagging

The team worked on describing important domain units in the Home Inventory System. This included identifying how items, alerts, restock rules, inventory states, and household interactions are represented within the system.

This task helped clarify the main vocabulary and structure used by the Alerts & Restock feature.

=== Glossary and Key Term Review

The team reviewed important terminology used throughout the project documentation. This helped make sure that terms such as inventory item, alert, restock rule, threshold, grocery list item, and notification are used consistently.

This is important because the Alerts & Restock module depends on clear definitions to avoid confusion during implementation and testing.

=== Human Behavior Analysis

The team analyzed how unpredictable human behavior can affect shared inventory systems. For example, different household members may update quantities at different times, forget to mark items as used, or manually restock an item without updating the app.

This analysis helped identify possible edge cases for alerts and restock automation.

=== Reactive Behavior Modeling

The team documented how the system should react when inventory conditions change. For example, when an item becomes low in stock, the system should trigger an alert and may also update the grocery list.

This task helped connect inventory events to automated system behavior.

=== Alerts Interface Usability Review

The team reviewed the Alerts interface using usability and heuristic evaluation principles. The purpose was to check whether users can clearly understand alerts, identify which items need attention, and respond to alerts easily.

This supports the goal of making the Alerts & Restock feature simple and useful for household users.

=== Cubit and State Management Analysis

The team analyzed the use of Cubit for state management in the Alerts UI. This focused on separating UI logic from business logic, making the alerts feature easier to maintain, test, and expand.

This supports better architecture for the Alerts & Restock module.

== Restock Automation Engine Progress

In addition to the completed documentation tasks, Team 2 also worked on the Restock Automation Engine.

The Restock Automation Engine is responsible for checking inventory conditions and deciding whether an automated restock action should be created.

The main logic includes:

[source,text]
----
- Checking item quantity
- Detecting low-stock items
- Detecting out-of-stock items
- Comparing current quantity against defined thresholds
- Creating or updating grocery list items
- Avoiding duplicate grocery list entries
----

The engine supports the automation flow between inventory tracking and grocery list generation.

=== Restock Automation Flow

[source,text]
----
Inventory item is updated
        |
        v
System checks current stock level
        |
        v
System compares quantity with threshold
        |
        v
If item is low or out of stock:
    Create or update grocery list entry
        |
        v
User can view restock item in grocery list
----

== Blockers and Challenges

=== Shared Household Behavior

One challenge is handling inventory updates from multiple household members. If two users update the same item close together, the system needs to use the most recent quantity and avoid creating incorrect alerts.

=== Duplicate Restock Actions

The automation engine must avoid creating duplicate grocery list entries when the same item is already listed.

=== Threshold Differences

Different item categories may require different threshold logic. For example, food items, cleaning supplies, and household products may not all use the same restock rules.
