== Main Contributions

=== Grocery List Display Work

The team worked on making grocery list items display on the page.

This feature is important because the Alerts & Restock module depends on users being able to see which items need to be restocked. When an item becomes low in stock or out of stock, the system should be able to create or update a grocery list item, and that item should appear clearly on the grocery list page.

Expected behavior:

[source,text]
----
Inventory item becomes low or out of stock
        |
        v
System detects restock condition
        |
        v
Grocery list item is created or updated
        |
        v
Grocery list page displays the item
----

=== Low-Stock Alert Trigger Analysis

The team worked on analyzing the conditions that trigger low-stock alerts in shared household inventories.

The main condition being documented is:

[source,text]
----
0 < current_quantity <= minimum_threshold
----

This means that an item should trigger a low-stock alert when its quantity is greater than zero but less than or equal to the minimum threshold. If the quantity reaches zero, the system should treat the item as out of stock instead of low stock.

This task helps clarify how alerts should behave when inventory quantities change.

=== Dependency Modeling Between Threshold Rules and Restock Actions

The team also worked on documenting how inventory threshold rules influence automated restock actions.

This analysis focuses on how item quantity, stock thresholds, and inventory status determine whether the system should create or update a grocery list item.

Basic dependency flow:

[source,text]
----
Inventory quantity changes
        |
        v
Threshold rule is evaluated
        |
        v
Low-stock or out-of-stock condition is detected
        |
        v
Automated restock action is triggered
        |
        v
Grocery list is created or updated
----

This documentation supports the design of the Restock Automation Engine.

=== Inventory Stock Movement Tracking

The team continued working on inventory stock movement tracking.

This task focuses on tracking how item quantities change over time. It supports the Alerts & Restock module because stock movement history can help explain why an item became low, empty, or restocked.

Examples of stock movement events include:

[source,text]
----
- Item quantity decreases after use
- Item quantity increases after restocking
- Item quantity reaches the low-stock threshold
- Item quantity reaches zero
- Item quantity changes because of a household member update
----

== Blockers and Challenges

=== Routing and Navigation Fixes

The task for adding the route and navigation link for the Alerts & Notifications screen is marked as needing a fix. This may block users from accessing the alerts screen properly until the routing issue is resolved.

=== Duplicate Grocery List Entries

The system needs to avoid creating duplicate grocery list entries when the same item is already listed. This is important for the Restock Automation Engine and grocery list display.

=== Shared Household Updates

Since multiple household members can update inventory items, the system must use the latest saved quantity before triggering alerts or restock actions.

=== Alert Consistency

Low-stock and out-of-stock alerts must be triggered correctly. The system should avoid repeated notifications for the same item when an alert is already active.

=== Documentation Alignment

The team needs to make sure all documentation, diagrams, and lecture topic tasks match the current implementation of the Alerts & Restock module.
