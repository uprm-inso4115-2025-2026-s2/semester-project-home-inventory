== Terminology

This section defines the domain’s ubiquitous language: preferred terms for entities, actions, events, and rules observed in student households. The definitions aim to stay independent of any specific system-to-be or implementation.

[cols="^,^,4,3",options="header"]
|===
|Term |Type |Definition |Synonyms / notes

|Household
|Entity
|A living unit (dorm, apartment, shared house) where one or more people manage household essentials.
|Previously “Home”.

|Household member
|Entity
|A person who belongs to a household and participates in consuming, replenishing, or coordinating around items.
|Previously “Owner” or “Occupant”.

|Location
|Entity
|A storage place within a household (kitchen cabinet, pantry area, bathroom closet, car trunk, hallway bin).
|Previously “Room”.

|Item type
|Entity
|The kind of item independent of a specific package (e.g., “Milk”, “Dish soap”, “Toilet paper”).
|Previously “Product”.

|Item instance (package)
|Entity
|A concrete package/unit of an item type (e.g., a specific milk carton or a specific box of medicine).
|Previously “Stock”.

|Inventory (reality)
|Entity
|The set of item instances that physically exist in the household at a given time.
|Real world, not a list.

|Tracked inventory record
|Artifact
|A representation of availability used by household members (fridge note, list, memory, chat message). It may diverge from reality.
|Supports “tracked vs reality”.

|Availability
|Property
|A household’s current understanding of “how much is left,” often qualitative (“full”, “almost empty”, “backup”).
|May later be represented as quantity.

|Quantity
|Property
|A more precise measure of availability (count/units). Quantity can be unknown or approximate.
|If used, it must never be negative.

|Low-stock judgment
|Event/Rule
|A decision that an item is running low, based on a household strategy (“feels light”, “opened backup”) or a threshold.
|Not inherently numeric.

|Low-stock threshold
|Rule/Property
|A boundary value used to decide when an item is considered low.
|May be fixed quantity initially.

|Out-of-stock
|State/Event
|The condition/event where an item is considered unavailable because it is gone (quantity at zero or judged empty).
|Distinct from low-stock.

|Backup supply
|Practice/Concept
|A household practice of keeping a spare instance/package (backup) to reduce the risk of running out.
|Often used instead of numeric tracking.

|Consumption
|Action
|Using an item over time, reducing availability.
|May vary by context (exam weeks, guests).

|Replenishment (restock)
|Action
|Acquiring more of an item, increasing availability.
|Often triggered after low/out/expiration.

|Restock list / planner
|Artifact
|A household-oriented list of items that should be replenished soon.
|Exists as a concept (notes/list).

|Alert / notification (domain-level)
|Artifact/Event
|A message that informs a household member about an actionable condition (low-stock, out-of-stock, expiring/expired, budget threshold crossed).
|Delivery method is an implementation choice.

|Snooze / defer
|Action
|A household member delays acting on an alert for a period of time.
|Domain intent: “remind me later”.

|Dismiss / ignore
|Action
|A household member acknowledges an alert and chooses not to act (at least for now).
|Should not cause repeated spam without changes.

|Expiration date
|Property
|A date after which an item instance is considered expired/undesirable (applies to some items, not all).
|May be unreadable/missing.

|Near-expiry
|State/Event
|An item instance is approaching expiration within a defined window (e.g., N days).
|Window definition belongs in requirements later.

|Expired
|State/Event
|An item instance is past its expiration date.
|Expired does not always imply “unavailable”; policy varies.

|Spending event / spending entry
|Record
|A record of a financial transaction in a household (purchase/refund/adjustment), optionally linked to an item type and category.
|Supports budgeting/reporting.

|Spending category
|Entity/Label
|A grouping for spending (e.g., groceries, cleaning, personal care).
|May align with item categories, but not required.

|Budget
|Entity/Rule
|A financial limit defined for a period (household-level or category-level) used to evaluate spending.
|Budget window/cycle is part of requirements.

|Budget cycle (period)
|Concept
|The time window over which spending is compared to a budget (weekly, monthly, custom).
|Resets/rollovers are policy decisions.

|Budget threshold crossing
|Event
|Spending reaches or exceeds a defined threshold (e.g., 80%, 100%).
|Can trigger alerts, subject to anti-spam rules.

|Transaction (quantity change)
|Record
|A record of an availability change (consume/restock/adjust) with time and reason.
|Supports traceability and verification.
|===
