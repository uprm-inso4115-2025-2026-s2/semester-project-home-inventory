

== Domain Invariants

Stable domain rules that hold across the household inventory domain.

=== Core invariants

* A household has one or more household members while it is active.
* An item instance belongs to exactly one household.
* An item instance is stored in some location, even if that location is forgotten or described informally.
* A household may contain multiple item instances of the same item type at the same time.
* A household member’s participation in a household may change over time.
* If a spending entry is recorded, it remains associated with the household in which the spending event occurred.
* If an item instance has an expiration date, it may become expired without being consumed.

=== Ownership

An item instance has exactly one owner. The owner is either:

* a *household member* (personal item — e.g., "my pasta sauce in the fridge"), or
* the *household itself* (shared item — e.g., "our kitchen towels").

One owner can own many items; one item has exactly one owner. When ownership is the household, every member of that household has access to the item through their membership.

When a household member is removed, their personal items may need to be reassigned or removed, but items owned by the household are unaffected. Spending entries remain attributed to the original payer regardless of ownership changes (DR-6).
