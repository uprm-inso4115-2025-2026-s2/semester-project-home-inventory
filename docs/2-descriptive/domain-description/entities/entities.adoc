== Entities

Key domain entities (nouns) and their attributes. These entities describe the domain independently of any specific system implementation.

[cols="2,4,4",options="header"]
|===
|Entity |Description |Key attributes

|Household
|A living unit where household essentials and spending are managed.
|name (optional label), members, locations, inventory (reality), optional budgets

|Household member
|A person who belongs to a household and consumes/replenishes items.
|name/identifier (domain-level), household reference, role (informal)

|Location
|A storage place within a household where items are kept.
|name/label, household reference, optional purpose

|Item type
|A definition of an item independent of a specific package/purchase.
|name, optional category, optional brand/size/variant, optional barcode

|Item instance (package)
|A concrete package/unit of an item type that exists in the household.
|item type reference, location reference, ownership (personal/shared), quantity/availability, optional purchase date, optional expiration date, optional notes

|Tracked inventory record
|A representation households use to reason about availability (may diverge from reality).
|artifact type (note/chat/list/memory), item reference (explicit or implicit), last-updated (if recorded)

|Restock list/planner
|A list of items judged to need replenishment soon.
|entries (item types), reason (low/out/expired), optional responsibility/intent marker, status (active/deferred)

|Alert / notification record
|A record of an alert-worthy condition presented to household members.
|type (low/out/near-expiry/expired/budget), target (item type/instance or budget), created time, read/unread, dismissed/deferred

|Spending entry
|A logged financial transaction event within a household.
|amount, date/time, payer (member), scope (shared/personal), optional item type reference, category, notes, transaction type (purchase/refund/adjustment)

|Budget
|A spending limit defined for a period.
|scope (household/category), limit amount, cycle (period), start/end (or recurrence), thresholds (warning levels), visibility rules

|Budget status (derived)
|A derived view of spending vs budget within a cycle.
|spent-to-date, remaining, percent used, threshold-crossed flags
|===
