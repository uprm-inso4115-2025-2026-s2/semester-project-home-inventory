== Events

Domain events are things that happen in the domain that matter to stakeholders (often phrased as past-tense verbs). These events are independent of any particular system-to-be, even if a later system represents or reacts to them.

[cols="2,4,4",options="header"]
|===
|Event |Description |Notes / examples

|Item acquired
|A household member obtains an item instance (possibly in bulk) and brings it into the household.
|“Bought 24 yogurts from Costco.”

|Item stored
|An item instance is placed in a household location.
|“Rice stored in kitchen cabinet; backup stored in hallway bin.”

|Item consumed
|A household member uses an item, decreasing its availability.
|Consumption rate may change during exams/guests.

|Backup opened
|A household member begins using the backup supply of an item.
|Often used as a qualitative low-stock signal.

|Low-stock judged
|A household member judges an item as running low.
|“Feels light” / “backup opened” / threshold-based.

|Out-of-stock noticed
|A household member discovers an item is gone when needed.
|“Toilet paper runs out unexpectedly.”

|Replenishment decided
|A household member decides the household should acquire more of an item type.
|May be prompted by low/out/expiration.

|Item replenished
|New item instance(s) of an item type are acquired to restore availability.
|Can be duplicated if coordination fails.

|Duplicate purchase occurs
|More of an item type is acquired unnecessarily because members acted on inconsistent information.
|Common for milk/eggs/rice per rough sketch.

|Item approaching expiration
|An item instance is within a near-expiry window (window definition later).
|Motivates timely decisions.

|Item expired
|An item instance passes its expiration date (if it has one).
|Policy varies by item type/household.

|Item discarded
|A household member throws away an item instance (expired/spoiled/unwanted).
|Can occur even when another unopened instance exists.

|Alert generated (domain-level)
|A low/out/near-expiry/expired/budget condition becomes actionable and is communicated to household members.
|Delivery method is not part of the domain event.

|Alert dismissed / deferred
|A household member acknowledges an alert and chooses to ignore it or act later.
|Supports “snooze” and anti-spam behavior.

|Restock entry created/updated
|An item is added to (or updated on) a restock list/planner due to low/out/expired conditions.
|May be manual or derived from judgments.

|Spending recorded
|A household member logs a spending transaction (purchase/refund/adjustment).
|May be shared or personal.

|Budget threshold crossed
|Spending in a budget cycle reaches a warning level or exceeds the limit.
|May trigger an alert.

|Budget cycle started/ended
|A new budget period begins or a period closes.
|Supports reset/rollover policies.

|Household membership changed
|A member moves in/out; household participation changes.
|Motivates access removal expectations later.
|===
