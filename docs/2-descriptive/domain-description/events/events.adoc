== 2.1.5 Events

[abstract]
Domain events are things that happen that matter to stakeholders (often past-tense verbs).

[cols="2,2,4,3",options="header"]
|===
|Event |Triggered by a |What changes |Related entities

|A purchase has been recorded. 
|Triggered by the User recording a purchase of an Item that they include in their User Inventory. 
|The purchase is now reflected in the Spending Tracker. 
|User, Spending Tracker, Item.

|Spending totals have changed. 
|Triggered by the recording of a purchase or the deletion of a recorded purchase. 
|The Spending Tracker now reflects updated spending totals. 
|User, Spending Tracker.

|An Item has been created. 
|Triggered by a User registering a new Item. 
|The User Inventory now contains a recorded Item, and the Item can be Searched/Filtered. 
|User, User Inventory, Item.

|An Item's metadata has been updated. 
|Triggered by a User modifying the metadata of an existing Item or the Item reaching its expiration date. 
|The Item's metadata is now updated, and the User Inventory reflects the updated information. 
|User, User Inventory, Item.

|A User Inventory has been generated. 
|Triggered by an addition of an Item to the User Inventory for the first time.
|The User Inventory now reflects the presence of a new Item.
|User Inventory, Item.

|Updated information has been reflected in the User Inventory. 
|Triggered by a modification of inventory contents. 
|The User Inventory now reflects the updated information. 
|User Inventory, Item.

|The recorded quantity of an Item in the User Inventory has increased. 
|Triggered by an addition of an already existing Item to the inventory. 
|The User Inventory now reflects an increase in the recorded quantity of an Item. 
|User Inventory, Item.

|An Item has been considered sufficiently stocked. 
|Triggered by a recognition of adequate availability of an Item. 
|The User Inventory now reflects the sufficient stock status of an Item. 
|User, User Inventory, Item.

|The recorded quantity of an Item in the User Inventory has decreased. 
|Triggered by a removal of an already existing Item from the User Inventory. 
|The User Inventory now reflects a decrease in the recorded quantity of an Item. 
|User Inventory, Item.

|An Item in the User Inventory has reached a low-stock condition. 
|Triggered by a recognition of an Item's quantity falling below a defined threshold. 
|The Item is flagged as low-stock in the User Inventory,and Item is included in the Restock Planner. 
|User, User Inventory, Item, Restock Planner.

|A low-stock alert has been issued.
|Triggered by a recognition of an Item reaching a Low-stock condition. 
|The User is alerted of the low-stock condition of an Item. 
|User, Item.

|An Item has neared its expiration date. 
|Triggered by a recognition of an Item nearing its expiration date. 
|The User is now alerted that an Item is nearing its expiration date, the User Inventory reflects the nearing expiration status. 
|User, Item, User Inventory.

|An Item has been expired. 
|Triggered by a recognition of an Item reaching its expiration date. 
|The User Inventory reflects the expired status.
|User, User Inventory, Item. 

|An expiration alert has been issued. 
|Triggered by a recognition of an Item becoming expired. 
|The User is alerted of the expired status of an Item. 
|User, Item.

|An Item has become an unavailable Item. 
|Triggered by a recognition of an Item becoming unavailable. 
|The User Inventory reflects the unavailability, and reflects the unavailable item in the Restock Planner. 
|User, User Inventory, Item, Restock Planner.

|A Restock Planner has been created. 
|Triggered by a compilation of Items that require replenishing. 
|The User now has a record of Items that need to be restocked. 
|User, User Inventory, Item, Restock Planner.

|An Item in the Restock Planner has been marked as fulfilled. 
|Triggered by a recognition of an Item being replenished. 
|The Restock Planner now reflects an updated status for the Item. 
|User, Restock Planner, Item.

|A Restock Planner has been updated.
|Triggered by changes in the status of Items that require replenishing. 
|The User now has an updated record of Items that need to be restocked (Restock Planner). 
|User, User Inventory, Item, Restock Planner.

|The User Inventory is now shared between multiple Users. 
|Triggered by an establishment of shared access to the User Inventory. 
|The User Inventory is now accessible to multiple Users, and reflects shared information, including the Restock Planner. 
|User, User Inventory, Restock Planner

|The shared relationship of the User Inventory has been revoked. 
|Triggered by any User revoking shared access to the User Inventory. 
|The User who revoked shared access to the User Inventory no longer sees updates done by other Users on the User Inventory, reflecting information specific to the individual User. 
|User, User Inventory, Restock Planner.

|An Inventory Summary Report has been generated. 
|Triggered by a User requesting an Inventory Summary Report. 
|The User now has access to a report summarizing the contents of a User Inventory. 
|User, User Inventory.

|A Spending Report has been generated. 
|Triggered by a User requesting a Spending Report. 
|The User now has access to a report summarizing their spending based on recorded purchases. 
|User, Spending Tracker, Items.

|===
