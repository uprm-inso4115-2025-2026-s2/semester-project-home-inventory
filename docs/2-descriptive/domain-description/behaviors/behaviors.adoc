== Behaviors

Recurring behaviors or rules in the domain: patterns, policies, and "how things are done."

=== Behavior patterns

* **Restock cycle:** A stakeholder consumes stock → quantity decreases → low-stock threshold is crossed → alert issued → purchase is made → quantity replenished.
* **Shared purchase flow:** A stakeholder buys a shared item → logs the purchase → the system splits cost across household members → each member sees their share.
* **Expiration monitoring:** Stock nears its expiration date → the system transitions the item to an _Expiring_ state → alert issued → stakeholder removes or discards the item → a waste event is logged.

=== Domain rules

* One barcode corresponds to exactly one Product definition (DR-1, DR-14, DR-17).
* A Stock duplicate entry submitted within 30 seconds triggers automatic quantity aggregation rather than a new entry (DR-7, IR-11).
* Spend Events are immutable — inventory corrections must never overwrite financial history (DR-3, DR-9).
* A Household must always have exactly one Owner (DR-4).
* Ownership must be transferred before an Owner can leave a Household (DR-4).

=== Edge cases

* If a stock entry is submitted while offline and synced later, the 30-second duplicate window is evaluated against the server-received timestamp, not the client timestamp.
* If an item has no expiration date, expiration monitoring is skipped for that stock entry.
* If an Owner attempts to leave without transferring ownership, the action is blocked until a new Owner is designated.
