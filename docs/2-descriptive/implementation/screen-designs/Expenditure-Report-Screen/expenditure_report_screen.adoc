== Overview

The Expenditure Report screen provides a visual breakdown of household spending by category
for a selected date range. It is accessible from the Reports Overview page via the
*Expenditures* card.

The screen is implemented at:
`src/lib/features/reports/presentation/pages/expenditure_report_page.dart`

Route path: `/home/reports/expenditures`

== Screen Layout

=== App Bar

* *Back arrow* ‚Äî navigates to the Reports Overview page.
* *Title* ‚Äî "Expenditures" (Poppins SemiBold 18 sp, black).
* *PDF export button* ‚Äî captures the pie chart and exports the full report as a PDF using `PdfExportService.exportExpenditureReport`.
* *Filters button* ‚Äî sage-green pill (accent color `#8B9D7F`) with a dropdown arrow;
  placeholder for future date-range and category filtering.

=== Report Metadata Header

Displayed beneath the app bar before the chart:

[source]
----
March 9 ‚Äì 15
2026
Total Spent: $221.00
----

* Date range derived from `ExpenditureState.startDate` / `endDate`.
* Total computed as the sum of all category amounts.

=== Pie Chart (donut style)

* Drawn with a `CustomPainter` (`_PieChartPainter`) ‚Äî no external charting library required.
* Each slice corresponds to one `ExpenditureCategory`, sized proportionally to its amount.
* A white hole at 50 % radius creates the donut effect and displays the total amount label
  in the centre (`#3A5A40` green, bold).
* A thin white stroke separates adjacent slices for visual clarity.
* Below the chart a `Wrap` legend shows a colour dot + category name for every slice.

=== Category Breakdown List

A white card below the chart showing one row per category with:

* Colour-coded dot matching the pie slice.
* Category name.
* Dollar amount (bold).
* Percentage of total spend.
* A linear progress bar indicating the share of total spend.

== Architecture

The screen follows the same BLoC pattern as `InventoryStockReportPage`:

[source]
----
ExpenditureReportPage (StatelessWidget)
  ‚îî‚îÄ‚îÄ BlocProvider<ExpenditureCubit>
        ‚îî‚îÄ‚îÄ _ExpenditureView (StatefulWidget)
              ‚îú‚îÄ‚îÄ _DateRangeHeader
              ‚îú‚îÄ‚îÄ RepaintBoundary (_PieChartCard)
              ‚îÇ     ‚îú‚îÄ‚îÄ _PieChartPainter (CustomPainter)
              ‚îÇ     ‚îî‚îÄ‚îÄ _PieChartLegend
              ‚îî‚îÄ‚îÄ _CategoryBreakdownList
                    ‚îî‚îÄ‚îÄ _CategoryRow (per category)
----

* `ExpenditureCubit` extends `Cubit<ExpenditureState>` and holds category data.
* `ExpenditureState` is immutable and exposes a `totalAmount` getter.
* `PdfExportService.exportExpenditureReport` was added to the existing service to handle PDF generation.

== PDF Export

Tapping the PDF icon in the app bar:

. Captures the pie-chart widget as a PNG via `RenderRepaintBoundary`.
. Calls `PdfExportService.exportExpenditureReport` with the state data and the captured image.
. Opens the system print/share dialog via the `printing` package.

The generated PDF contains:

* Report title and date range.
* Generation timestamp.
* Total spent amount.
* Embedded pie-chart image (if capture succeeded).
* A category breakdown table (Category | Amount | % of Total).
