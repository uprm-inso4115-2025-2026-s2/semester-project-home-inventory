=== Duplicate Prevention Implementation

==== Database Constraints (Supabase)

All duplicate checks must be executed *before* insert. Recommended constraints:

* `UNIQUE(barcode)` on the `products` table
* `INDEX(normalized_name, normalized_brand, normalized_size)` on the `products` table

==== Supabase Query Strategy

===== Barcode Check

[source,sql]
----
SELECT id FROM products
WHERE barcode = :barcode
LIMIT 1;
----

===== Structured Metadata Match

[source,sql]
----
SELECT id FROM products
WHERE normalized_name = :normalized_name
AND normalized_brand = :normalized_brand
AND normalized_size = :normalized_size
LIMIT 5;
----

===== Similarity Check (Optional / Advanced)

[source,sql]
----
SELECT id, similarity(normalized_name, :normalized_name) AS score
FROM products
WHERE similarity(normalized_name, :normalized_name) > 0.85
ORDER BY score DESC
LIMIT 5;
----

NOTE: Similarity search must NOT auto-merge — it may only suggest candidates.

==== Flutter Responsibilities

* Normalize all product fields (lowercase, trim, collapse spaces, convert units) *before* sending to backend.
* Display duplicate warning modals when the backend returns a match.
* Require explicit user confirmation for: near-duplicate reuse and merge actions.

==== Supabase Responsibilities

* Enforce `UNIQUE(barcode)` and reject conflicting inserts.
* Store normalized fields in dedicated columns (`normalized_name`, `normalized_brand`, `normalized_size`).
* Execute duplicate check queries atomically before any insert transaction.
