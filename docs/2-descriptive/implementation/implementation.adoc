== 2.3 Implemented End-to-End Features

This section documents the current implementation status of the repo's core features based on the existing code. Each feature is verified against actual code paths and component wiring to show what is currently present, what is scaffolded, and what remains planned.

**Status Legend:** Completed | In Progress | Planned

---

=== Feature 1: Core Inventory Management (F-INV)

**User Value:** Supports manual item capture, inventory categorization, and item editing. This feature is the most complete part of the current repo.

**Primary Users:** Household managers, kitchen coordinators

**Current Implementation:**
- `src/lib/features/core_inventory/presentation/pages/inventory_page.dart` provides the main inventory screen.
- `src/lib/features/core_inventory/presentation/pages/add_item_page.dart` and `edit_item_page.dart` provide item entry/edit forms.
- `src/lib/features/core_inventory/presentation/widgets/item_form.dart` captures item name, details, quantity, and expiration date.
- `src/lib/config/injection_dependencies.dart` registers `InventoryCubit`, Supabase client, data sources, repositories, and use cases.
- `src/lib/features/core_inventory/presentation/cubits/inventory_cubit.dart` implements loading, adding, updating, and deleting stock.
- Data sources and repositories are defined in `src/lib/features/core_inventory/data/` for Supabase-backed inventory and product operations.

**Actual Code Paths:**
- Main route: `src/lib/core/presentation/routes.dart` connects `/inventory` to `InventoryPage`.
- Inventory category navigation and add/edit routing are defined in `src/lib/features/core_inventory/presentation/routes.dart`.
- Add / edit item flows are implemented as form-only pages with `ItemForm`.

**Implementation Notes:**
- The inventory feature currently uses manual form entry only. There is no barcode scanning UI or barcode plugin integration in the current code.
- The main inventory page currently displays category cards and navigates to category pages; it does not yet present a full item list with Supabase-backed data on the route shown.
- `InventoryCubit` exists and is registered, but the current inventory pages do not yet wire the cubit into visible state management on screen.

**Completion Status:** In Progress — core UI flow scaffolding is present, and backend data layer wiring exists, but full inventory data rendering and barcode scanning are not fully implemented.

=== Feature 2: Automated Expiration Alerts & Notifications (F-ALERT + F-AUTOCHECK)

**User Value:** Intended to notify users proactively about expiring items and reduce food waste.

**Primary Users:** All household members, household admin

**Current Implementation:**
- Alert-related domain logic is present in `src/lib/features/alerts_restock/domain/usecases/`:
  - `check_expiration.dart`
  - `check_stock.dart`
  - `automated_check_expiration.dart`
  - `automated_check_stock.dart`
- These files provide pure business logic for classifying expiration and stock status.

**Actual Code Paths:**
- There is an Alerts tab route in `src/lib/core/presentation/routes.dart`, but it currently serves a placeholder page only.
- No alert center screen, deep link handling, or notification handler is wired into the active route set.
- There is no evidence of Supabase cron edge functions, FCM/APNs push notification integration, or `alert_log` storage in the current repo.

**Implementation Notes:**
- The code contains domain models for expiration and stock checks, but this logic is not connected to a UI flow or backend notification pipeline.
- Any completed alert generation and dispatch functionality is not present in the current workspace.

**Completion Status:** Planned — domain use cases exist, but end-to-end alert delivery and UI integration are not implemented.

=== Feature 3: Grocery List Management (F-GL-AUTO + F-GL-USER)

**User Value:** Provides a shopping list interface and supports manual grocery item creation.

**Primary Users:** Shoppers, household managers

**Current Implementation:**
- `src/lib/features/grocery_list/presentation/pages/home_page.dart` implements the Grocery tab with tabs for `Collections` and `My Grocery List`.
- `src/lib/features/grocery_list/presentation/pages/my_grocery_list_page.dart` shows a static list of grocery items using `ItemTile` widgets.
- `src/lib/features/grocery_list/presentation/pages/add_item.dart` provides a manual add-item form for custom grocery items.
- Routing is defined in `src/lib/features/grocery_list/presentation/routes.dart` for grocery home, categories, custom items, add item, and history.
- The grocery feature currently uses `TodoCubit` from `src/lib/features/example_feature/presentation/cubits/todo_cubit.dart` as its active state provider.

**Actual Code Paths:**
- Grocery home route: `/grocery_home` in `src/lib/core/presentation/routes.dart`.
- Add item route: `/grocery_home/add_item`.
- Custom item route: `/grocery_home/categories/custom_items/edit_item`.

**Implementation Notes:**
- The current grocery list screen is mostly static and does not contain an auto-generated shopping list or restock workflow.
- There is no evidence that `automated_check_stock` or `automated_check_expiration` is linked to grocery list generation.
- Items are rendered via UI widgets, but the checked-off/restock dialog flow is not implemented in the current code.

**Completion Status:** In Progress — grocery UI scaffolding exists, but automatic list generation and restock integration remain unimplemented.
