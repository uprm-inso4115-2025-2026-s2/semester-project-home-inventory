=== Quick Reference Summary


. Product duplicates:
* Never create a new Product for barcode matches -- always reuse.
* Text matches suggest the existing product; user can override with explicit confirmation.
. Stock duplicates:
* 0-30s: auto-merge quantity and block new entry.
* 30s-2m: warn and let user choose merge vs separate entry.
* >2m: allow separate entry (still suggest merge when signals are obvious).
. Shared households:
* Shared inventory can merge counts, but spending attribution stays per user event.
. Budget/reporting:
* Inventory state changes must not rewrite spend history.
. UX:
* Clear prompts, minimal friction, undo available, consistent across all flows.
