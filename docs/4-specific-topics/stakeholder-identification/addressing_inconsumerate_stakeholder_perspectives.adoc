= Notification System Design: Addressing Incommensurate Stakeholder Perspectives
:author: Devlin Hahn

**AI Disclosure:** Portions of this design were developed with the assistance of Artificial Intelligence tools. The analysis of incommensurate perspectives, notification defaults matrix, and user stories were developed by the author based on the existing persona definitions and stakeholder mapping. AI tools were used to format the document. All content has been reviewed and verified by the human author, @devlin-hahn04.

== Executive Summary

Four distinct user personas: Mateo (roommate coordination), Sofía (family management), Gerardo (student/learning focus), and Adriana (occasional user), hold fundamentally incompatible perspectives on how notifications should function. As noted in the stakeholder analysis presentation, "stakeholder perspectives may (in part) be incommensurate" (slide 27), meaning these views cannot be satisfied by a single uniform solution.

The proposed solution uses persona-based onboarding defaults, granular notification categories with independent controls, and progressive disclosure to maintain simplicity for users who need it. This approach ensures completeness of concerns while maintaining transparency throughout development.

== 1. Analysis of Incommensurate Perspectives

=== 1.1 Notification Conflict

The four personas hold divergent views on notifications:

[options="header"]
|===
| Persona | Stakeholder Group | Perspective on Notifications
| Mateo | End User (Roommate) | Requires frequent push notifications for shared item updates, roommate activity alerts, and regular inventory review reminders. Notifications are essential for household coordination.
| Sofía | End User (Homeowner/Renter) | Needs reliable but selective notifications. Push for urgent matters (warranty expirations), email digests for non-urgent updates. Cannot tolerate frequent interruptions during family time.
| Gerardo | End User (Student/learning focus) | Wants learning-focused notifications—tips, tutorials, and feature suggestions. Prefers in-app messages over push notifications. Sees notifications as educational tools.
| Adriana | End User (Occasional User) | Desires no notifications. Uses the app only 1-2 times per year for insurance purposes. Finds push notifications intrusive and will uninstall if bothered.
|===

=== 1.2 Why These Perspectives Are Incommensurate

These perspectives are fundamentally incompatible for three reasons.

**First, notification frequency expectations conflict.** Mateo wants real-time updates; Adriana wants zero interruptions. A single frequency setting cannot satisfy both. Defaulting to frequent notifications drives Adriana away; defaulting to minimal notifications makes the app useless for Mateo.

**Second, the perceived purpose of notifications differs.** For Mateo, notifications enable household coordination. For Gerardo, they provide educational value. For Adriana, they are nuisances. These different purposes cannot be served by identical notification content.

**Third, delivery channel preferences conflict.** Mateo wants push notifications for immediacy. Sofía wants a mix of push and email digests. Gerardo prefers in-app messages. Adriana wants email only. No single channel satisfies all.

=== 1.3 Consequences of Prioritizing One Persona

Prioritizing Mateo causes Adriana to uninstall due to notification fatigue. Prioritizing Adriana causes Mateo to abandon the app for shared spreadsheets. Averaging the perspectives leaves both dissatisfied: Mateo receives insufficient notifications to be useful; Adriana receives more than she wants and still uninstalls.

A configurable solution is required rather than a single uniform design.

== 2. Onboarding Classification

During initial setup, users select their primary use case:

....
How do you plan to use Home Inventory?

Please select the option that best describes your primary use case:

☐ Shared household with roommates
   I'll track items shared with others and want to stay updated on
   what everyone adds.

☐ Family household management
   I'll manage inventory for my family and want reliable reminders
   for important dates like warranty expirations.

☐ Learning and organizing
   I'm new to inventory management and want tips on how to organize
   effectively.

☐ Simple personal inventory
   I just want a basic list of my belongings for insurance purposes.
   I don't want frequent notifications.
....

Selection maps to personas and default profiles:

[options="header"]
|===
| User Selection | Persona | Default Notification Profile
| Shared household with roommates | Mateo | Push-heavy, real-time updates for collaborative features
| Family household management | Sofía | Mixed push/email, selective urgency for warranties and deadlines
| Learning and organizing | Gerardo | In-app learning content, digest for non-educational updates
| Simple personal inventory | Adriana | Minimal, email-only essential reminders (annual insurance review)
|===

== 3. Notification Categories and Defaults

=== 3.1 Categories

[options="header"]
|===
| Category | Description
| Shared Item Activity | Alerts when a roommate adds, edits, or removes items in shared spaces
| Inventory Review Reminders | Prompts to review and update inventory (quarterly/annual)
| Collaborative Activity Feed | Summaries of activity in shared inventory spaces
| Learning Tips & Tutorials | Suggestions for better organization, feature guides
| App Updates & Announcements | New features, important notices, maintenance updates
|===

=== 3.2 Defaults Matrix

[options="header"]
|===
| Category | Mateo Default | Sofía Default | Gerardo Default | Adriana Default
| Shared Item Activity | Push (immediate) | Digest (daily) | Push (immediate) | Off
| Inventory Review Reminders | Push (quarterly) | Push (quarterly) | Digest (quarterly) | Email (annually)
| Collaborative Activity Feed | Push (real-time) | Digest (weekly) | Push (real-time) | Off
| Learning Tips & Tutorials | Off | Off | In-app (on login) | Off
| App Updates & Announcements | Digest (monthly) | Digest (monthly) | Digest (monthly) | Digest (monthly)
|===

=== 3.3 Channel Definitions

[options="header"]
|===
| Channel | Description
| Push | Mobile push notification, appears on lock screen and in notification center
| Digest | Summary delivered via in-app notification center; optional email delivery
| In-app | Visible only when user opens the application; no external alert
| Email | Email-only delivery, no push notification
| Off | No notifications for this category
|===

=== 3.4 Justification for Defaults

**Mateo (Shared household with roommates):**
Shared Item Activity and Collaborative Activity Feed set to Push (immediate) because real-time awareness is critical for household coordination. Inventory Review Reminders set to Push (quarterly) to maintain organization without overwhelming. Learning Tips set to Off because Mateo is already experienced with organization.

**Sofía (Family household management):**
Shared Item Activity set to Digest (daily) to stay informed without constant interruptions during family time. Inventory Review Reminders set to Push (quarterly) to maintain accurate records.

**Gerardo (Learning and organizing):**
Learning Tips set to In-app (on login) to provide educational value when engaged. Shared Item Activity and Collaborative Activity Feed set to Push (immediate) to learn through observation of shared spaces.

**Adriana (Simple personal inventory):**
All push notifications Off by default. Only essential reminders via Email: Inventory Review Reminders (annually). App Updates set to Digest to maintain awareness of important changes without intrusion.

== 4. Settings Interface Design

=== 4.1 Progressive Disclosure Approach

The settings interface uses progressive disclosure: all users see a simple view with profile-based defaults. Advanced settings are hidden behind an expandable section. This balances Mateo's need for granular control with Adriana's need for simplicity.

=== 4.2 Simple View

The simple view shows:
* Current notification profile with option to change
* Each notification category with a single selection (channel only)
* "Show Advanced Settings" button to expand

=== 4.3 Advanced View

Expanding advanced settings reveals:
* Radio buttons for channel selection per category
* Frequency/timing dropdowns where applicable
* "Hide Advanced Settings" button to collapse

=== 4.4 Design Justification

**Profile-based defaults** reduce configuration burden
**Simple view** hides complexity
**Advanced view** provides granular control when needed
**Profile change** updates defaults while preserving manual overrides
**Consistent layout** maintains usability across both views

== 5. User Stories

[options="header"]
|===
| Persona | User Story
| Mateo | As Mateo, I want to receive push notifications whenever a roommate adds or edits a shared item, so that I always know what belongs to whom without having to constantly check the app.
| Sofía | As Sofía, I want to receive email digests for non-urgent updates but push notifications for warranty expirations, so that I stay informed without being interrupted during family time.
| Gerardo | As Gerardo, I want to see in-app tips and tutorials about inventory organization when I log in, so that I can learn better ways to manage my belongings over time.
| Adriana | As Adriana, I want to receive no push notifications and only get an annual email reminder to review my inventory, so that the app never bothers me when I'm not actively using it.
|===

== 6. Alignment with Stakeholder Mapping

The design aligns with the stakeholder mapping phases:
* Onboarding classification captures user needs during requirements engineering (Phase 2)
* Requirements are documented so developers can implement without constant user input during implementation (Phase 3)
* All four stakeholder groups are explicitly addressed, ensuring completeness of concerns

== 7. Conclusion

The four personas hold incommensurate perspectives on notifications. Mateo requires frequent real-time updates; Adriana requires zero interruptions. These views cannot be reconciled through a single uniform solution.

The proposed solution resolves the conflict through:
* Persona-based onboarding defaults
* Granular notification categories with independent controls
* Progressive disclosure in the settings interface
