=== Objective

Identify edge cases involving duplicates and budgeting conflicts to proactively address ambiguous scenarios that could cause incorrect spending calculations, inaccurate reports, or inconsistent roommate coordination.

Scope: cases that occur *after* duplicate detection signals exist (barcode match, text similarity, same product metadata, recent add/import patterns), focused on system behavior to protect budget integrity.

Out of scope:

* The duplicate detection algorithm itself (see: Duplicate Item Criteria)
* Full budgeting feature design (this document defines conflict rules only)

=== Decision Rules Matrix (Quick Reference)

[cols="1,2,2,2", options="header"]
|===
| Scenario | Inventory Action | Budget Action | UI Requirement

| Same barcode
| Reuse Product; apply stock duplicate time-window rules
| Create spend event for add (if provided)
| No confirmation for product reuse

| Same name, different size
| Separate Product by default
| Normal spend event linking
| Near-duplicate prompt with size highlight

| Same product, different expiration
| Separate stock entries
| Spend unchanged
| Warning if user tries to merge

| CSV imported twice
| Prevent duplicate spend via fingerprint
| Skip duplicate spend events by default
| Import review summary

| Delete duplicate inventory entry
| Remove inventory only
| Keep spend unless explicitly excluded
| Two-step confirm: inventory vs budget exclusion

| Refund
| Decrement inventory (if returned)
| Create negative spend event
| Refund flow required

| Shared household merge
| Merge inventory counts only
| Spend attribution stays per-user
| "Who bought this?" prompt with confirmation if assigning to another member

| Expired item removal
| Remove from inventory; log waste event
| Spend unchanged
| Optional "Add to waste report" checkbox (default on)
|===

=== Safe Defaults (Quick Reference)

* Prefer preserving data granularity (separate entries) over aggressive merging when uncertain.
* Never auto-merge Spend Events.
* Never auto-delete Spend Events as a side-effect of inventory correction.
* Inventory auto-merge actions must support undo (10 seconds).
